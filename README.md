# official-home-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本项目主要用于存放和展示一系列独立的 HTML 页面。这些页面可能代表不同时期、不同用途或不同风格的静态内容，通过统一的仓库进行管理和版本控制，便于维护、回溯与分享。

仓库本身不针对任何特定域名或网站，仅作为静态页面的集合与发布源。

## 目录说明

```
.
├── pages/               # 存放所有 HTML 页面文件
│   ├── example1/
│   └── example2/
├── assets/              # 公共资源文件（CSS、JS、图片等）
├── index.html           # 可选：仓库首页或导航页
└── README.md            # 本文件
```

- `pages/`：每个子目录通常对应一个独立页面，包含其所需的 HTML、样式和脚本。
- `assets/`：存放多个页面可能共用的资源，如通用样式表、图标或字体文件。
- `index.html`：如果存在，可作为仓库内容的入口或总览页。

## 页面归档说明

本仓库中的页面按以下原则进行归档：

- 每个页面拥有独立的文件夹，避免文件冲突。
- 页面之间相互独立，不依赖其他页面内容（公共资源除外）。
- 归档时建议保留原始文件结构，便于后续维护和更新。
- 不包含任何动态后端逻辑，所有页面均为纯前端静态内容。

## 维护说明

- 新增页面：在 `pages/` 下创建新文件夹，放入 HTML 文件及相关资源。
- 更新页面：直接修改对应文件夹内的文件，建议附带 commit 说明变更内容。
- 删除页面：移除整个文件夹，并在 commit 中注明原因。
- 公共资源：如需要修改全局样式或脚本，请谨慎评估影响范围。

欢迎通过 Issue 或 Pull Request 提出改进建议。请确保提交的内容不包含敏感信息、商业推广或违反平台政策的内容。
